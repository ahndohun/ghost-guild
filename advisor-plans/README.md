## Findings considered and rejected

- Make the whole interface bright and high-saturation: rejected because it
  conflicts with the documented dark 16-bit colosseum direction. Strong color
  belongs on primary actions, rewards, danger, and selected state.
- Create 110 unrelated perk illustrations immediately: rejected because it
  multiplies production and consistency risk. Use a composable icon vocabulary
  for attack, defense, movement, economy, behavior, and signature effects,
  combined with class color and tier framing.
- Preserve the current responsive single-column document: rejected because it
  directly contradicts DESIGN.md's fixed letterboxed game viewport and worsens
  selection overload on narrow screens.
- Treat passing TestSprite visibility checks as proof of UI quality: rejected
  because the current plans do not assert geometry, readability, comparison
  feedback, or information uniqueness.
