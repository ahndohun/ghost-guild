# AI 대개편 계획 — 히어로·몬스터 이동 지능 (AI Overhaul Plan)

보드 리뷰(2026-07-11 새벽): "시작하면 다들 오른쪽으로 달리고, 충돌판정이 없다시피 해 몬스터와 겹치고, 이유 없이 벽에 비빈다. 경우에 따라 맵 중앙을 지키는 게 합리적일 수도 있다 — 최적 알고리즘으로 보는맛을 높이자."

4개 지적 전부 코드 수준에서 원인을 확정했다. 관전이 코어 루프인 게임에서 이동 AI의 질이 곧 제품의 질이다.

## 1. 진단 — 증상별 확정 원인

**① "시작하면 다들 오른쪽으로"** — `src/sim/movement.ts` `chooseDirection`: 16방향 유틸리티 스코어링에서 동점일 때 첫 후보가 이긴다. 첫 후보 = `movementDirections[0]` = 각도 0 = **동쪽(오른쪽)**. t=0에는 모든 신호가 0(초기 moveDir=(0,0)이라 히스테리시스 0, 적은 스폰 직후 원거리, 전리품 없음, 중앙이라 벽 반발 0) → 전 히어로가 매 런 오른쪽 러시. 결정론이라 100% 재현.

**② "몬스터와 겹침"** — 어떤 개체 쌍에도 분리(separation)·겹침 해소(push-out)가 없다. 적 AI(`src/sim/enemies.ts` `tickEnemies`)는 최근접 히어로 직선 추적뿐이라 적끼리 한 점에 포개져 "한 마리처럼 보이는 덩어리"가 되고, 히어로-적도 이동 적용 후 위치 해소 패스가 없어 그대로 관통·중첩된다.

**③ "이유 없이 벽에 비빔"** — 두 겹 원인. (a) `wallRepulsion`이 소프트 가중(×80)뿐이라 카이팅 벡터가 벽을 향하면 "벽에 미끄러지는 방향"이 최고점이 되고, 히스테리시스(직전 방향 ×4)가 그걸 고착시킨다. (b) 탈출 장치인 progress guard가 **"적이 140px 내에 없을 때"만** 작동한다(`updateProgressGuard`) — 추격당하며 벽에 몰린 상황(적이 가까움)에서는 앵커가 계속 리셋돼 영원히 발동하지 않는다. 게이트가 정확히 거꾸로다.

**④ "중앙을 지키는 게 합리적일 수도"** — 스코어에 공간 개념이 없다. 개방도(코너에 몰리기 전에 피하기), 중앙 통제, 탈출로 확보 같은 항이 0. 위협의 본질은 최근접 적이 아니라 **포위·코너링**인데 그걸 평가하지 않는다.

## 2. 처방 — 컨텍스트 스티어링 2맵 구조로 승격

현 16방향 단일 가산 스코어를 업계 표준 **context steering(관심 맵 + 위험 맵 분리)**로 승격한다 (Andrew Fray, "Context Steering", Game AI Pro 2 — gameaipro.com 공개 챕터; 오픈 아레나에서 경로탐색 없이 결정론·저비용으로 지역 최소점을 피하는 정석). 원리: 방향별로 **위험 맵**(적 근접·벽·포위)과 **관심 맵**(교전 거리·전리품·전략 자세)을 따로 쌓고, 위험이 임계 이상인 방향은 **하드 마스크**(관심이 아무리 높아도 제외) 후 남은 방향에서 관심 최대를 고른다. "위험을 뚫고 전리품에 끌려가는" 아티팩트가 구조적으로 사라진다.

### W1. 버그 소거층 (관전 품질의 최소선, ~2.5h)

1. **동점 파괴 + 유휴 기본 행동**: 신호가 없을 때의 기본 관심 = **아레나 중앙 링으로의 전개**. 히어로별 결정론 각도 슬롯(hero.id 유도)으로 링 위 자기 위치를 갖는다 — 시작 장면이 "전원 우측 러시"에서 "각자 자리로 산개"로 바뀐다. 아레나 모드(4히어로)에서 즉시 보는맛.
2. **벽 하드 마스크**: 룩어헤드(속도×0.5s) 안에 경계를 넘는 방향은 위험 맵에서 차단. 소프트 반발은 유지하되 "벽으로 미끄러지는 최고점"이 성립 불가능해진다.
3. **progress guard 게이트 수정**: "적 근접 시 리셋"을 제거하고 **변위 기반**으로 단순화 — 최근 2.5초 순변위 < 20px이면 적 유무와 무관하게 발동, 탈출 방향은 중앙이 아니라 **개방도 최대 방향**(아래 W2의 개방도 항 재사용).
4. **겹침 해소 패스** (이동 적용 후 위치 교정, id 오름차순 반복 = 결정론):
   - 적↔적: 반경 합 미만이면 절반씩 밀어냄(보이드 분리) — "덩어리 몬스터" 소멸, 무리가 무리로 보임.
   - 히어로↔적: 히어로는 통과 대신 접촉 반경에서 저지·미끄러짐(접촉 데미지 규칙 불변 — 판정은 기존 radius 그대로).
   - 히어로↔히어로(아레나): 소프트 분리 — 4명의 라이벌이 한 점에 포개지는 현상 제거.
   - 성능: 적 300+ 대비 공간 해시 버킷(셀 32px) — O(n²) 회피, 결정론 유지(버킷 순회도 id 정렬).

### W2. 전략층 — "최적 알고리즘"과 보는맛 (~2h)

5. **개방도(openness) 항**: 방향별로 "그쪽으로 갔을 때의 탈출로 수"를 평가(적 원뿔 투영 + 벽 거리). 코너에 몰리기 **전에** 빠져나오는 움직임이 생긴다 — 관전자가 "얘 잘한다"라고 느끼는 니어미스 회피의 원천.
6. **전략 자세를 기질과 연결** (class-identity-plan P1.5(a)와 같은 레버 — 같은 사이클로 묶을 것):
   - **guardian/holdGround = 중앙 홀드**: 보드 지적 그대로 — 중앙 점유가 관심 맵의 기본값인 클래스. 중앙은 모든 방향으로 탈출로가 있는 최적 지점이라 "지키는" 그림이 나온다.
   - duelist/kiter = **링 순회(orbit kiting)**: 무지성 후퇴 대신 중앙 주위 링을 도는 카이팅(스티어링이 자연히 만드는 원운동) — 뱀서 숙련자의 움직임 문법.
   - hoarder = 전리품 동선 최적화(개방도 마스크 안에서).
7. **의도 자막 노출**: `currentIntentReason`이 이미 심에 있다 — 명패/커멘터리(P4 배너 채널)로 표면화("코너 탈출!", "중앙 사수"). 관전 이해도가 오르면 같은 AI도 더 똑똑해 보인다.

### W3. 몬스터 AI 다양화 (~1.5h, bestiary-plan B와 결합)

8. kind별 이동 시그니처(전부 무RNG — 위상은 enemy.id 유도):
   - condemned(느린 무리): 분리력 강화로 **넓게 포위하는 무리** — 직선 스택 대신 초승달 포위.
   - hound(빠름): **오빗-런지** — 히어로 주위를 돌다가 주기 위상에서 직선 돌진. 맹수답다.
   - murmillo(탱크): 등속 직진 + 밀리지 않음(분리 가중 최소) — 중장보병의 위압.
   - elite/champion: 돌진 전 0.5s 텔레그래프(정지+렌더 예고) — HoT의 "예고 공격" 문법, 회피 장면 연출.

## 3. 리스크·결정론

- 전 변경 무RNG(위상·슬롯은 id 유도) — seeded 결정론 유지. **골든 전면 갱신 필수**(이동이 바뀌면 모든 골든이 깨진다 — 의도 변경, 커밋 메시지 명시).
- 밸런스 파급 大: 생존시간 분포가 바뀐다. 기존 property 테스트(비전투 정지 방지·클래스 상이 결과) + 5클래스×2시드 생존시간 스모크로 가드, 급변 시 웨이브 테이블 미세 보정.
- **파일 소유권 주의**: `movement.ts`는 class-identity-plan P1.5(a)와 이 계획 W2-6이 같은 파일을 만진다 — **병렬 금지, 같은 워커·같은 사이클로 통합 실행**.
- 리플레이 호환: 전투는 (seed, loadouts) 재계산이라 코드 버전만 일치하면 무결 — 서버 스키마 무변경.

## 4. 수용 기준

1. **시작 10초**: 솔로·아레나 각 1런 — 오른쪽 러시 소멸, 아레나 4히어로가 서로 다른 방향으로 산개.
2. **겹침**: 30초 관찰에서 적끼리 완전 중첩(한 마리로 보임) 0회, 히어로가 적 스프라이트 위에 겹쳐 서 있는 프레임 0회.
3. **벽**: 180초 런에서 "벽에 붙은 채 제자리 유지" 3초 이상 지속 구간 0회(도주 중 스침은 허용).
4. **중앙 전략**: knight(guardian)로 60초 관찰 — 중앙 반경 1/3 안 체류 시간이 과반. mage/thief는 링 순회가 관찰될 것.
5. **보는맛 블라인드**: 개편 전/후 30초 클립 비교 — 후자가 "AI가 잘한다"로 읽히는지(니어미스 회피·포위 탈출 장면 존재).
6. 게이트: typecheck·vitest(골든 갱신)·build → 배포 → tsloop → LOOP 기록. 결정론 2회 재현 테스트.

## 5. 실행 편성

컷 라인: **W1이 최소 출고선**(보이는 버그 3종 소거 — 우측 러시·겹침·벽 비비기). W2가 보드가 말한 "최적 알고리즘 + 보는맛"의 본체. W3는 bestiary-plan과 결합해 차기 사이클 가능. 총 ~6h. 우선순위는 4대 계획 통합 편성(HANDOVER §7-0) 기준으로: 이 계획의 W1은 무엇보다 먼저 — **버그로 보이는 것은 콘텐츠보다 먼저 고친다.**
