# 로비 IA/UX 재설계 계획 (UI-IA Redesign Plan)

보드 리뷰(2026-07-11, 주석 스크린샷 2장): ① 이름 입력창이 직관적이지 않다 — 로비 진입 전에 받아라 ② Overview 탭이 무의미 — Class와 통폐합 ③ Training·Gear 배치 재고, Gear는 Inventory로 개명 ④ 기록 배지(Best 123s)가 생뚱맞다 ⑤ 하단 버튼이 의미 없이 나열, DEPLOY라는 단어가 어색 — 전투 느낌 + 솔로/아레나 통합 후 모달 선택 ⑥ 우하단 버튼들은 설정으로 — REPLAY COACH는 왜 있는지 모르겠다 ⑦ 전반적으로 IA가 엉망.

## 1. 근본 진단

화면이 **기능이 생길 때마다 패널·탭·버튼을 덧붙이는 방식**으로 자랐다. 방문 빈도와 의사결정 순서 기준의 정보 구조가 없다. 플레이어의 실제 루프는 셋뿐이다: **본다(내 검투사 상태) → 키운다(골드 소비: 클래스/트리/훈련/장비) → 싸운다(출전)**. 화면 위계가 이 루프를 그대로 반영해야 한다.

개별 확인 사항:
- REPLAY COACH의 정체 = 온보딩 코치마크(COACH 1/4 "Choose a class" 단계 안내) 다시 보기. 기능은 정당(처음 보는 저지에게 유용)하나 **이름이 기능을 전달하지 못한다** — 보드조차 용도를 못 알아봤다.
- 이름 입력(`player-name` 텍스트필드)이 로비 상단에 상시 노출 — "언제든 고치는 설정값"처럼 보이지만 실제로는 한 번 정하는 정체성이다.
- Best 123s 배지가 골드 옆에 부유 — 골드는 재화(소비 맥락), 기록은 검투사의 서사(정체성 맥락). 다른 맥락의 정보가 한 자리에 있다.

## 2. 재설계 — 화면 골격

```
┌──────────────────────────────────────────────┐
│ (우상단 고정) 골드 배지 · ⚙설정 아이콘          │
│ [스테이지] 검투사 스프라이트 + 명패              │
│   명패 = 이름 · 클래스 · 기질 · 최고기록 N초     │  ← Overview·기록 배지 흡수
│ [가로 3탭] CLASS & TREE │ TRAINING │ INVENTORY │  ← 골드 소비처(키운다)
│ [탭 콘텐츠 영역]                                │
│ ██████████ TO BATTLE! (단일 대형 버튼) ██████████ │  ← 루프의 다음 행동(싸운다)
└──────────────────────────────────────────────┘
```

위계 원칙: 시각 크기 1위 = 출전 버튼(다음 행동), 2위 = 스테이지(내 검투사), 3위 = 육성 탭. 설정·사운드류는 아이콘으로 강등.

### R1. 이름 입력 → 최초 1회 온보딩 (보드 ①)
- 타이틀 PRESS START 직후, 세이브에 이름이 없으면 **DQ풍 이름 입력 모달**("그대의 이름을 남겨라 — 모래가 기억할 것이다"). 확정 후 로비 진입.
- 로비 상단 텍스트필드 제거 → 명패의 이름 클릭 시에만 rename 팝오버(저빈도 기능은 숨긴다).
- `player-name` testid는 모달 입력으로 이전. 관련 클라우드 테스트(이름 지속성 등) 스텝 정합 확인·재생성.

### R2. 탭 통폐합 4→3 (보드 ②③)
- **Overview 폐지**: 내용(개요·기록)은 스테이지 명패로 흡수. 선택 클래스 상세는 Class 탭의 본문이 된다 — 중복 화면 소멸.
- **Gear → INVENTORY 개명** (보드 지시 그대로).
- 좌측 세로 탭 → **스테이지 아래 가로 3탭**: CLASS & TREE / TRAINING / INVENTORY. 세 탭 모두 "골드를 쓰는 곳"이라는 단일한 존재 이유를 갖는다 — 탭줄 옆에 골드 잔액이 상시 보이는 배치가 정합.

### R3. 기록 배지 재배치 (보드 ④)
- Best Ns 배지 삭제 → 명패 세 번째 줄("최고 기록 123초 생존"). 기록 갱신 순간에는 결과 화면에서 강조(이미 best-survival 표기 존재).

### R4. 출전 단일 버튼 + 모드 모달 (보드 ⑤)
- DEPLOY SOLO / DEPLOY ARENA 두 버튼 → **대형 단일 버튼 "TO BATTLE!"** (전투 어휘 — DEPLOY는 군사 배치 용어라 검투사 세계와 부정합).
- 클릭 시 모달: **"연습전 — 홀로 모래 위에 선다"(솔로) / "흥행전 — 세계의 고스트 3인과 겨룬다"(아레나)** + AUTO-RUN 토글(연속 출전 옵션을 출전 맥락 안으로).
- testid 이전 맵: `deploy-solo`·`deploy-arena`는 모달 내 버튼으로 유지(셀렉터 보존으로 클라우드 테스트 파손 최소화), 단일 버튼은 `battle-open` 신규. tsloop 플랜 중 출격 경로 언급 테스트 전수 확인.

### R5. 설정 팝오버 (보드 ⑥)
- 우상단 ⚙ 아이콘 → 팝오버: SOUND 토글 · **TUTORIAL 다시 보기**(구 REPLAY COACH — 기능을 말하는 이름으로 개명) · (여유 시) BGM/SFX 분리.
- 하단 바에서 SOUND·REPLAY COACH·AUTO-RUN 제거 — 하단은 출전 버튼 전용.

## 3. 수용 기준

1. **3초 테스트**: 로비 도착 직후 "다음에 뭘 눌러야 하는가"가 3초 안에 읽힌다 — 출전 버튼이 시각 위계 1위인지 스크린샷으로 판정.
2. **클릭 수**: 재방문 유저 기준 로비→전투 시작 2클릭(TO BATTLE → 모드 선택). 신규 유저는 이름 모달 포함 3클릭.
3. 탭 3개 각각의 존재 이유를 한 줄로 설명 가능("전부 골드 소비처").
4. 로비에 상시 노출되는 텍스트 입력 필드 0개.
5. E2E 정합: testid 이전 맵 문서화(DESIGN §9 갱신) + 영향받는 클라우드 테스트 확인·재생성 + tsloop 통과까지가 완료 정의.
6. 게이트: typecheck·vitest·build → 배포 → tsloop → LOOP 기록.
